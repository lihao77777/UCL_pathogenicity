Pathogenicity

Pathogenicity is a utility, running via cluster and taking data from scoreassoc or a table to produce plots and scripts using different ways to show the results of pathogenic variances.

There is no formal published account of pathogenicity but the actions it performs were based on the idea from this publication, which should be cited:

Curtis, D. Assessment of ability of AlphaMissense to identify variants affecting susceptibility to common disease. Eur J Hum Genet (2024). https://doi.org/10.1038/s41431-024-01675-y

Broadly, pathogenicity takes a list of variances and the output of variance scores to produce three results. By taking original data from scoreassoc or a filtered table, pathogenicity could show the predicted pathogeny for different prediction methods across one protein via plots.

Besides, it can also produce a file which could then be used in PyMol to visualise the mutation points on a specific protein with control and cases.

Additionally, another file produced by pathogenicity could allow the position of pathogenic variance to be visualised on gene sequence via Protein Paint.

Running Pathogenicity

Typical usage:

/share/apps/R-3.6.1/bin/Rscript /home/lihaotao/scripts/Pathogenicity20250120.R --read_file /home/lihaotao/saofiles/UKBB.HT.forAnnot.20231112.INPPL1.txt --gene_name INPPL1  --pred_method AM_score --pred_method MutPred_ --pred_method VEST4_ra  --pred_method CADD_raw --transcript NM_001567.4 --write_table INPPL1_filtered_table --write_plot_1 INPPL1_plot_1 --write_plot_2 INPPL1_plot_2 --write_script_pymol INPPL1_pymol --write_script_pp INPPL1_pp --set_wd /home/lihaotao/scripts/

The code is programmed in R language and should be applied via cluster. Therefore, Rscript should first be opened in the cluster before writing any commands. As temporary files will be generated and then read for producing scripts.

User can use one of two types of input files, including files from scoreassoc or a filtered table, to produce mainly three results, plots, a script for PyMol, and a script for Protein Paint through applying pathogenicity. The arguments for the program can be specified on the command line, and user can apply different files and obtain different results through using different commands. The arguments are in pairs, with the first member of the pair, starting with --, specifying the option to be set and the second member the value for that option.

For the simplest usage, the code could be used to show what is the predicted pathogeny for each variance across a protein. The pathogenic results for different prediction methods via cases and controls could also be raised simutaneouly and shown via another plot.

Besides, user can choose to produce a new file which could be used in PyMol to visualise the pathogenic points in a protein 3D structure. To use the PyMol, users should first download the appropriate PDB file, open it in PyMol, and then copy the script produced by xxx in the PyMol command line.

Moreover, pathogenicity can produce file used in Protein Paint for visualising the pathogenic points in a protein linear gene sequence by cases and control. To apply the script, users need to find the correspond gene sequence in Protein Paint. After check whether the transcript numbers are the same for Protein Paint and their input in pathogenicity commands, users then could copy the content in the produced script and paste in Protein Paint to see the pathogenic results.

In order to run pathogenicity you will need the following:

For input files, user should decide whether they want to use a file or a table to start with. For files, the files should at least contain locus, contAA, contAB, caseAA, caseAB, transcript, comment, and pathogenic scores for prediction methods. Users should provide information including locus, contAB, caseAB, transcript, aa_number (amino acid number), aa_change (abbreviation for amino acids change), and pathogenic scores for prediction methods as these data is necessary to generate plots and scripts.

 

Each gene name and its transcript number should be recorded. There might be different transcripts for a same protein gene and users should only one consistently.

 

Each time, although different prediction methods could be analysed only one protein gene should be used.

 

When pathogenicity is run it extracts the target protein data from previously processed files, either a file or a table, annotates and coordinates them. Then it uses ggplot to plot files, showing the difference between different prediction methods. It also generates three other new files which could be further applied to use in Pymol and Protein Paint.

Typically a parameter file specified by --read_file would contain the specifications for the analyses and the locations of the data and reference files and then the other --argument argument would specify some traits of the protein to be analysed.

All the arguments are in pairs. The first member of the pair must begin with a double-hyphen and the second member provides the value for the argument. The arguments can be provided on the command line in argument files or a combination of both. Arguments are separated by any amount of white space. If an argument value includes spaces then it must be enclosed in single quotes. For arguments on the command line, the operating system will apply the normal command line substitutions, which may affect arguments including asterisks and/or double quotes, so these should either be within an argument file (probably easiest) or else enclosed in single quotes to prevent substitution.

Here is the list of options:

 

--read_file pathname

Specifies the path of a file (e.g. proteinx_file.txt) from which to read additional arguments. If users decide to use a table rather than a saofile as input, they should leave a blank in the value space. Once this file has been read successfully then further options will be read from the command line:

xxx --…/proteinx_file.txt

 

--set_wd path access

Specifies the path you would like to access

 

--gene_name genename

Provides the name of the targeted gene.

 

--transcript name

Specifies the name of the transcript for the protein.

 

--write_table table name

Specifies the name for later generated filtered table.

 

--write_plot_1 plot name

Specifies the name for later generated plot1, showing pathogenic variances.

 

--write_plot_2 plot name

Specifies the name for later generated plot2, showing controls vs cases.

 

--write_script_pymol name

Specifies the name of txt generated for Pymol use.

 

--write_script_pp

Specifies the name of txt generated for ProteinPaint use.

 

--predmethod name

Provide names of interested prediction methods. Write –predmethod before each target prediction method. If there are four prediction (e.g. method1, method2, method3, method4) methods then write in commands:

--predmethod method1 --predmethod method2 --predmethod method3 --predmethod method4

The names of the prediction methods written have to be exactly the same as those from the datasaofile to ensure they could be successfully recognised.

 

List of operators/functions

 

Binary operators/functions (placed between values/arguments)

= or IS

Return 1 if numerical values of arguments are equal or if both are identical strings, e.g. "Fred" = "Tim" is false (0),  and using the NOT operator NOT ("Fred" = "Tim") is true (1).

!=

Return 1 if numerical values of arguments are not equal (NB false if two different strings both have numerical value 0, e.g. "Fred" != "Tim" is false (0).

<=

Returns 1 if first argument is less than or equal to second, else 0.

< 

Less than operator, etc.

>=

 

> 

| or OR

OR operator, return 1 if either argument is true, only evaluate second if first is false.

& or AND

AND operator, return 1 if both arguments are true, only evaluate second if first is true.

IFTRUE

Return second argument if first is true, else return empty string "".

,

Comma operator - evaluate both arguments, return second.

-

Subtract second argument from first or if one argument, negate it.

+

Addition.

MOD

Modulo operator, 5 MOD 3 returns 2.

/

Division.

*

Multiplication.

 

 

It is possible to construct quite complicated expressions whose results are then used by pathogenicity and when it is run normally one cannot see the intermediate stages of the expression evaluation. If one specifies --debug 1 then the intermediate values for each part of the expression are output as they are evaluated and this can be helpful for trouble-shooting.
